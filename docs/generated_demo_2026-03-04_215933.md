**最小技术架构**
- 客户端：`Web`（React/Vue 任一）只连你自己的后端，不直连 Binance。
- 后端 API：`Node.js (Nest/Express)` 或 `Python (FastAPI)`，负责聚合 Binance 公共数据、缓存、限流、统一返回格式。
- 市场数据接入：
  - `REST`：拉取快照（行情、K线、24h统计）
  - `WebSocket`：订阅实时数据（ticker、depth、trade）
- 缓存层：`Redis`（可选但强烈建议），做短 TTL 缓存 + WS 最新值缓存。
- 数据存储：`PostgreSQL`（先存基础历史K线/订阅配置/系统日志）；最小可先不落库，仅缓存。
- 调度与任务：后端内置 `cron/queue`（如 BullMQ/Celery）定时补历史K线。
- 部署：`Nginx + API服务 + Redis + Postgres`，容器化用 `Docker Compose` 即可。

**后端任务拆解**
- Binance 连接层
  - 封装 REST Client（`/ticker/price`、`/klines`、`/ticker/24hr`）
  - 封装 WS Client（自动重连、心跳、断线恢复、订阅管理）
  - 统一异常映射（超时、429、网络抖动）
- 业务 API 层
  - `GET /symbols`：交易对列表
  - `GET /ticker?symbol=BTCUSDT`：最新价/24h信息
  - `GET /klines?symbol=BTCUSDT&interval=1m&limit=500`
  - `GET /depth?symbol=BTCUSDT&limit=20`
  - `GET /stream/ticker`（SSE/WS 转发给前端）
- 性能与稳定性
  - 请求限流（对前端和对 Binance 双向限流）
  - 缓存策略（ticker 1~3s，depth 500ms~1s，klines 10~30s）
  - 熔断与降级（上游异常时返回最近缓存 + 状态标记 `stale=true`）
- 可观测性
  - 结构化日志（请求耗时、Binance状态码）
  - 指标（QPS、错误率、WS重连次数）
  - 健康检查 `/health`（含 Binance 连通性）

**前端任务拆解**
- 页面最小集
  - 行情看板：交易对搜索 + 实时价格 + 24h涨跌
  - K线页：图表（TradingView Lightweight Charts/ECharts）
  - 深度/成交简版：买卖盘前20档 + 最新成交滚动
- 数据交互
  - 首屏走 REST 拿快照
  - 实时更新走后端 SSE/WS
  - 断线重连与“数据延迟”提示
- 状态管理
  - `React Query/SWR` 管 REST 拉取与缓存
  - WS/SSE 增量更新合并到本地状态
- 体验与容错
  - 骨架屏 + 错误态 + 空数据态
  - 交易对切换时取消旧请求，避免脏数据覆盖

**数据层任务拆解**
- 最小数据模型（先够用）
  - `symbols`：symbol、base、quote、status
  - `kline_1m`（或通用 kline 表）：symbol、open_time、OHLCV
  - `ingest_jobs`：任务状态/最后同步时间
  - `system_logs`：关键错误与告警
- 数据流
  - 实时：Binance WS → 后端内存/Redis → 前端订阅
  - 历史：定时 REST 补K线 → Postgres（用于回看与统计）
- 质量与一致性
  - K线按 `(symbol, interval, open_time)` 去重
  - 补数任务支持断点续传（按最后时间戳续拉）
  - 时间统一 UTC，前端本地化展示

**建议的 2 周最小里程碑**
- 第1-2天：后端 Binance REST/WS 封装 + `/ticker` `/klines`
- 第3-4天：前端行情页 + K线图首版
- 第5-6天：Redis 缓存、限流、重连、错误处理
- 第7-8天：历史K线入库任务 + 基础表结构
- 第9-10天：监控日志、健康检查、部署脚本
- 第11-14天：联调、压测、修复、文档

**最小接口清单（建议先实现）**
- `GET /api/market/symbols`
- `GET /api/market/ticker?symbol=BTCUSDT`
- `GET /api/market/klines?symbol=BTCUSDT&interval=1m&limit=500`

- `GET /api/stream/ticker?symbol=BTCUSDT`（SSE）或 `/ws`

如果你愿意，我可以下一步直接给你一版“可落地的目录结构 + 表结构SQL + OpenAPI 草案（可直接开工）”。
