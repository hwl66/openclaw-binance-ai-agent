## Agent 回复 2

```text
下面给你一个可落地的 **2周 MVP 计划**，目标是上线“**币安用户的 AI 风险预警 + 资产体检**”最小可用版本，先跑通“可解释、可触达、可验证”的闭环。

**MVP目标（2周后）**
- 对用户给出两类核心能力：`风险预警`（短期异常风险）+ `资产体检`（组合健康度与改进建议）。
- 覆盖币安现货/资金账户核心数据，支持日级刷新 + 关键事件准实时（分钟级）触发。
- 输出可解释结论：风险分、风险原因、建议动作（减仓/分散/止损提示等）。
- 有基础效果评估：命中率、误报率、用户点击/采纳率。

---

**一、数据源设计（MVP范围）**

- **用户账户与持仓（币安 API）**
  - 账户余额、资产分布、持仓数量与成本（若可得）、历史成交（买卖时间/价格/数量）。
  - 资金划转、充提记录（可选，MVP可先不做复杂链路归因）。
- **市场行情（币安行情接口）**
  - K线（1m/5m/1h/1d）、最新价、成交量、波动率、振幅。
  - 盘口深度简化指标（买卖价差、深度不平衡，可选）。
- **资产元数据**
  - 币种分类（主流币/山寨币/稳定币）、是否高波动标签、流动性等级（按日均成交额分层）。
- **衍生风险特征（内部计算）**
  - 用户层：集中度（Top1/Top3占比）、杠杆暴露（若接合约）、换手率、追涨杀跌特征、回撤。
  - 市场层：短时暴跌、放量下跌、异常波动、相关性突变。
- **标签与评估数据（离线）**
  - 事件标签：未来24h是否发生 `组合回撤 > X%`、`单资产跌幅 > Y%`、`波动异常`。
  - 用户行为标签：是否点击预警、是否执行建议（卖出/减仓/分散）。

> MVP建议先聚焦：`现货账户 + 前100流动性币种 + 日内/日级风控`，避免一开始接入过多复杂账户类型。

---

**二、模型方案（可快速上线）**

采用“**规则引擎 + 轻量机器学习 + LLM解释层**”三层结构，确保2周可交付。

- **1）风险预警模型（分类/评分）**
  - **规则层（高优先级）**：用于高精度触发  
    - 例：Top1持仓>60% 且该币24h波动>阈值；组合24h回撤已达预警线；短时放量下跌。
  - **ML层（XGBoost/LightGBM）**：预测未来24h风险事件概率  
    - 输入：用户持仓结构 + 行为特征 + 市场特征  
    - 输出：风险概率 `P(risk_24h)`，映射成 0-100 风险分。
  - **融合策略**：`最终风险分 = max(规则分, ML分)` 或加权融合（MVP建议max，简单可靠）。

- **2）资产体检模型（健康评分）**
  - 定义5个子分（每项0-100）：  
    - `集中度`（过于单一扣分）  
    - `波动暴露`（高波动资产占比高扣分）  
    - `流动性健康`（低流动性占比高扣分）  
    - `稳定资产缓冲`（稳定币/低波资产不足扣分）  
    - `回撤韧性`（历史最大回撤高扣分）
  - 总分加权汇总，输出等级：优秀/良好/警惕/高风险。
  - 每个扣分项给出可执行建议（例如“将Top1从72%降至45%以内”）。

- **3）解释与文案层（LLM，可选上线）**
  - 不让LLM直接决定风险，只负责把结构化结果转成用户可读解释。
  - 输入：触发规则、特征贡献（SHAP前3）、建议模板。
  - 输出：`风险原因 + 证据 + 建议动作 + 风险免责声明`。
  - 好处：提升可读性与点击率，同时可控合规。

---

**三、2周排期（按天）**

- **第1-2天：需求冻结与数据打通**
  - 定义风险事件标签与阈值（如24h组合回撤>5%）。
  - 打通币安账户与行情数据采集，建最小特征表。
  - 确定MVP币种池（如Top100）。

- **第3-4天：特征工程与规则引擎**
  - 完成集中度、波动、回撤、换手等核心特征。
  - 实现首版规则预警（可配置阈值）。
  - 离线回放最近30-90天数据做 sanity check。

- **第5-6天：ML模型训练与校准**
  - 训练LightGBM/XGBoost二分类模型。
  - 做时间切分验证（避免泄露），产出AUC/PR曲线。
  - 风险分校准（Platt/Isotonic可选，MVP可先分桶）。

- **第7天：资产体检评分卡完成**
  - 实现5维子分 + 总分 + 等级映射。
  - 完成建议模板（规则化，可先不用LLM）。

- **第8-9天：服务化与API**
  - 产出两个接口：`/risk-alert`、`/asset-health-check`。
  - 支持定时批量计算 + 事件触发计算。
  - 增加审计日志（输入快照、输出分数、触发原因）。

- **第10天：前端/消息触达**
  - App站内卡片或消息模板：风险等级、原因、建议。
  - 加点击埋点与采纳行为埋点（后续KPI评估关键）。

- **第11-12天：灰度与阈值调优**
  - 小流量灰度（例如5%用户），观察误报与投诉。
  - 调整阈值：优先控制高风险误报。

- **第13-14天：复盘与上线准备**
  - 输出KPI看板、案例复盘、已知问题清单。
  - 确认正式MVP发布与下一阶段需求。

---

**四、KPI设计（MVP必须可衡量）**

- **模型效果KPI（离线+在线）**
  - `风险事件召回率`（Recall@TopK 或 Recall at alert threshold）≥ 60%（MVP目标）
  - `预警精确率`（Precision）≥ 25%（先可用，后续优化）
  - `AUC` ≥ 0.70（二分类基线可接受）
  - `误报率`（无后续风险却频繁预警）周环比下降

- **业务KPI（用户侧）**
  - 预警消息点击率 CTR ≥ 8%
  - 体检报告查看率 ≥ 15%
  - 建议采纳率（如减仓/分散）≥ 5%
  - 被预警用户未来7天大回撤比例，相对对照组下降（核心价值指标）

- **系统KPI（工程侧）**
  - 日批成功率 ≥ 99%
  - 关键事件预警延迟 < 5分钟
  - 接口P95延迟 < 500ms（读缓存场景）

**五、MVP交付物清单**

- 数据：用户特征表、市场特征表、标签表（可回放）。
- 模型：规则引擎配置 + ML模型文件 + 风险分映射文档。
```

```text
**最小技术架构**

- 后端：`Node.js + Fastify`（或 `Python + FastAPI`）提供统一 API 层，负责拉取币安数据、做轻量清洗、缓存与限流。

  - `Redis`：短周期缓存（ticker/orderbook），削峰与抗抖。
  - `PostgreSQL`：落库 K 线、交易对元数据、任务状态，支持历史查询。
- 前端：`React + Vite`（或 `Next.js`）做行情页面、K线图、交易对搜索与基础指标展示。
- 调度与采集：
  - 实时：币安 `WebSocket`（如 `aggTrade`、`kline`、`depth`）。
  - 补偿：定时任务拉 REST（防 websocket 断线漏数）。
- 部署：1 台应用机（后端+前端）+ 1 台数据库（PG+Redis）即可起步；后续再拆。

**后端最小模块拆解**

- `binance-client`：封装 REST / WS（重连、心跳、退避、错误码映射）。
- `market-service`：统一业务接口（ticker、depth、klines、symbols）。
- `cache-service`：Redis key 设计与 TTL 策略（如 `ticker:BTCUSDT` 3~5 秒）。
- `ingest-worker`：订阅 WS，落内存队列，批量写 PG。
- `api-gateway`：对前端输出稳定接口，隐藏币安字段差异。
- `observability`：日志、基础指标（请求量、WS 重连次数、延迟）。

建议最小 API（给前端）：
- `GET /api/symbols`
- `GET /api/ticker?symbol=BTCUSDT`
- `GET /api/depth?symbol=BTCUSDT&limit=20`
- `GET /api/klines?symbol=BTCUSDT&interval=1m&start=&end=`
- `GET /api/health`

**数据模型（最小可用）**

- `symbols`
  - `symbol`(PK), `base_asset`, `quote_asset`, `status`, `updated_at`
- `klines`

  - 索引：`(symbol, interval, open_time desc)`
- `ticker_snapshot`（可选，若要回溯盘口/价格快照）
  - `symbol`, `ts`, `last_price`, `bid`, `ask`, `volume`

Redis（示例）：
- `ticker:{symbol}` JSON, TTL 3s
- `depth:{symbol}:{limit}` JSON, TTL 1s

图表建议：`TradingView Lightweight Charts`（轻、快、够用）。

**开发任务拆解（按迭代）**

- 迭代 0：工程初始化（0.5~1 天）
  - 建仓库结构：`apps/web`、`apps/api`、`apps/worker`
  - 环境变量、lint、基础 CI、Docker Compose（pg/redis）
```
