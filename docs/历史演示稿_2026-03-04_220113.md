  - `Backend`：Node.js (NestJS/Express) 或 Python (FastAPI)

  - `Scheduler`：后端定时任务（每5~60秒拉一次）
  - `Observability`：基础日志 + 健康检查 + 错误告警（先用日志即可）

---

**后端任务拆解**
- `API 接口封装`
  - 封装币安公开端点：`/exchangeInfo`、`/ticker/24hr`、`/depth`、`/klines`
  - 统一超时、重试、错误映射（429/418/5xx）
- `服务层`
  - `MarketService`：交易对列表、最新价、24h统计
  - `KlineService`：按 symbol/interval/time range 拉取K线
  - `DepthService`：盘口快照（可只做 top N）
- `缓存策略`
  - 热点数据 TTL：ticker 2~5s，depth 1~2s，exchangeInfo 1h
  - Cache Key 规范：`binance:ticker:{symbol}` 等
- `限流与熔断`
  - 对前端调用加本地限流（如 IP + 路径）
  - 对币安请求加令牌桶/并发上限，避免触发权重限制
- `标准化输出`
  - 后端统一字段（小数精度、时间戳单位 ms）
  - 响应里附带数据来源时间 `sourceTs`
- `定时任务（可选但推荐）`
  - 周期拉取并落库常用 symbol 的1m/5m K线
  - 失败重试与死信记录（最小可先写 error log）
- `健康与运维`
  - `/health`、`/ready`
  - 关键日志：外部API耗时、错误率、缓存命中率

---

**前端任务拆解**
- `页面最小集`
  - 行情总览页：交易对、最新价、涨跌幅、成交额

- `数据接入`
  - 仅调你自己的后端，不直连币安（便于统一限流和风控）
  - 轮询策略：ticker 2~5s，K线按 interval 刷新
- `图表与交互`
  - K线图（lightweight-charts / echarts）
  - 时间周期切换：1m/5m/15m/1h/4h/1d
  - symbol 搜索与收藏（先本地存储）
- `状态与容错`
  - Loading/空态/错误态
  - API 超时提示与自动重试（指数退避）
- `性能最小优化`
  - 列表虚拟化（交易对多时）
  - 防抖搜索、按需加载图表组件

---

**数据层任务拆解**
- `表结构（最小）`
  - `symbols`：symbol、base_asset、quote_asset、status、filters(JSONB)
  - `klines`：symbol、interval、open_time、open/high/low/close/volume、close_time
  - `jobs`：task_name、last_run_at、status、error
- `索引设计`
  - `klines(symbol, interval, open_time)` 唯一索引
  - `symbols(symbol)` 唯一索引
- `入库策略`
  - 幂等 upsert（按 symbol+interval+open_time）
  - 批量写入（降低 DB 压力）
- `保留策略`
  - 先保留近 30~90 天 1m K线（按存储成本调整）
  - 历史归档可后续补
- `数据质量`
  - 时间连续性检查（缺口告警）
  - 数值合法性检查（high>=low、volume>=0）
