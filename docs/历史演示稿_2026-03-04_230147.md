- `服务层`：统一业务接口（比如 `/api/markets`、`/api/kline`、`/api/ticker`），隐藏币安字段差异。

  - 高频接口（ticker、深度）短缓存 1-3s；

- `数据补齐任务`：每 1-5 分钟检查最近K线是否连续，不连续则回拉补齐。
- `可观测性`：接入结构化日志（请求ID、symbol、耗时、状态码）、Prometheus 指标（QPS、错误率、cache hit）。

**前端任务拆解**

  - K线图（可切周期）  
  - 告警配置（价格阈值）
- `数据接入`：首屏 REST 拉取快照 + WS 增量更新（防止白屏和跳变）。
- `状态管理`：按 symbol 维度管理行情状态，断线重连后自动重订阅。
- `交互与体验`：加载骨架屏、错误重试、最后更新时间标记。

  - 唯一键：`(symbol, interval, open_time)`
  - 缺口检测：按时间窗口检查连续性
- `保留策略`：
