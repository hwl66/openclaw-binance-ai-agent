**最小技术架构**
- 客户端（Web）：展示行情、K线、交易对列表，调用你自己的后端接口，不直连币安。
- 后端 API（单体服务即可）：统一封装 Binance REST/WebSocket，做限流、缓存、降级、字段标准化。
- 数据层（Redis + 可选 PostgreSQL）：Redis 做实时缓存；PostgreSQL 只在你需要历史分析/回放时启用。

**后端任务拆解**
- `BinanceClient` 封装
  - REST：`/exchangeInfo`、`/ticker/24hr`、`/klines`、`/depth`。
  - WS：`@trade`、`@bookTicker`、`@kline_1m`（按需）。
  - 统一超时、重试（指数退避）、错误码映射。
- 领域接口（给前端）
  - `GET /api/symbols`：交易对列表 + 状态 + 精度。
  - `GET /api/tickers`：全市场 24h 摘要（支持 topN/quote 过滤）。
  - `GET /api/klines?symbol=BTCUSDT&interval=1m&limit=500`
  - `GET /api/depth?symbol=BTCUSDT&limit=100`
  - `GET /api/stream`（SSE 或 WebSocket）推送标准化实时数据。
- 平台能力
  - 限流与防抖：避免前端频繁直打 Binance。
  - 缓存策略：热点接口 1~3 秒缓存；`exchangeInfo` 可 10~30 分钟缓存。
  - 熔断降级：Binance 抖动时返回最近缓存并标记 `stale=true`。
  - 时钟统一：全部转毫秒时间戳 + UTC。
- 安全（公开 API 场景）
  - CORS 白名单、基础 WAF 规则、请求频率限制。
  - 不存私钥（因为是公开 API），配置只含 base URL、超时、限流参数。

---

**前端任务拆解**
- 页面最小模块
  - 交易对搜索/切换（BTCUSDT 等）。
  - 价格卡片（最新价、24h 涨跌幅、24h 成交额）。
  - K线图（1m/5m/1h 切换）。
  - 深度简表（买一卖一、点差）或最近成交列表。
- 数据交互
  - 首屏走 REST 拉快照；进入页面后订阅 `/api/stream` 做增量更新。
  - 断线重连（指数退避），重连后自动补一次快照。
  - 本地状态分层：`snapshot` + `realtime patch`，避免全量重渲染。
- 体验与容错
  - 加载骨架、错误提示、`数据延迟` 标识。
  - 当后端返回 `stale=true` 时，UI 显示“使用缓存数据”。
  - 移动端优先保留：列表 + 价格卡 + K线，深度可折叠。

---

**数据层任务拆解**
- Redis（建议必选）
  - Key 设计：`ticker:{symbol}`、`klines:{symbol}:{interval}`、`depth:{symbol}`。
  - TTL：ticker 2s、depth 1s、klines 10~30s（按业务调）。
  - Pub/Sub（可选）：后端多实例间广播实时更新。
- PostgreSQL（最小可选）
  - 仅当你要历史查询/回测：落 `klines_1m`、`trades` 聚合表。
  - 表先少建：`market_klines(symbol, interval, open_time, ohlcv...)`。
  - 分区与归档可后置，MVP 阶段先保证可查可用。
- 数据质量
  - 字段标准化：价格/数量统一字符串或 decimal 处理，避免浮点误差。
  - 去重：按 `symbol + eventTime + tradeId/openTime` 幂等写入。
  - 时序对齐：K线以 `openTime` 为主键，不用本地接收时间。

---

**推荐最小技术栈（务实版）**
- 后端：Node.js + NestJS/Express（二选一）+ `ws` + Redis。
- 前端：React + Vite + 轻量图表库（`lightweight-charts`）。
- 数据：Redis（必选），PostgreSQL（可选）。
- 部署：单机 Docker Compose（`web` + `api` + `redis`）。
